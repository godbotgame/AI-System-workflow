## 🔀 Git 分支策略

| 分支类型 | 命名规范 | 用途 |
|----------|----------|------|
| 主分支 | `main` | 生产环境代码 |
| 开发分支 | `dev` | 集成开发代码 |
| 功能分支 | `feature/[module]-[task-id]` | 单个功能任务 |
| 修复分支 | `fix/[issue-id]-[desc]` | Bug 修复 |
| 发布分支 | `release/v[X.Y.Z]` | 阶段发布 |

**合并规则**：
- `feature/*` → `dev`（完成 Stage 5 验收后合并）
- `dev` → `main`（所有阶段完成后合并）
- 所有合并需通过 PR（Pull Request）
